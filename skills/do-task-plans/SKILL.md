# 执行计划

## 流程

对每个任务：
- 读取计划文件，当计划指示时引用相关仓库说明或技能
- 严格遵循每个步骤
- 按指定要求运行验证，不要跳过验证
- 没有新的验证证据，不要声称任务已完成或删除任务文件。
- `plan.md` 只是整体架构指南，任何情况下都不要修改、清理或记录进度到 `plan.md`。
- 如果无法运行验证，只在对应任务文件中将任务保持为 `已阻塞`，并记录跳过的检查，不要声称成功。
- 验证证据在最终回复中报告后，立刻删除该任务 `.md` 文件；不要修改 `plan.md`，不要记录其他东西，避免产生大量垃圾干扰其他任务。

## 何时停下并寻求帮助

**以下情况立即停止执行：**
- 遇到阻碍（缺少依赖、测试失败、指令不明确）
- 计划存在关键缺口，无法开始
- 你不理解某条指令
- 验证反复失败

**宁可请求澄清，也不要猜测。**
**不要强行突破阻碍** - 停下来提问。

## 何时回溯到前面的步骤

**以下情况返回审查流程：**
- 伙伴根据你的反馈更新了计划
- 基本方法需要重新思考
