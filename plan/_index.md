## 执行原则

- 只添加有助于理解模块职责、复杂流程、边界条件和公共 API 的中文注释。
- 避免逐行翻译代码，避免解释显而易见的赋值、导入和简单分支。
- 保持现有代码行为、格式和测试语义不变。
- 涉及 MCP、installer、解析器、resolver、watcher 等关键链路时，优先注释设计意图和失败场景。
- 每完成一批目录后运行适当的 `cargo test` 或更小范围测试。
